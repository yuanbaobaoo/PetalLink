# PetalLink 概要设计文档

> 版本：1.3.0 · 2026-07-13
> 技术栈：Tauri 2.x · Rust 后端 · Vue3/TS 前端
>
> **1.3.0 可靠性重构（2026-07-13）**：
> - 传输队列升级为持久化状态机：Pending / Running / WaitingForNetwork / BackingOff / VerifyingRemote / RestartRequired / Completed / Failed / Canceled；网络恢复、人工重试和启动恢复共用 TaskRunner。
> - 云端索引升级为 `tree + path_to_id + cursor` 原子可信检查点：全量采用“扫描前 cursor → 无副作用 BFS → Changes replay → fsync/rename 提交”；未追平时禁止上传覆盖、双向删除、reconcile 和 purge。
> - 上传恢复不再推算服务端偏移或把 update 降级 create；下载使用版本 sidecar + HTTP Range；所有失败态均不推进最后成功同步基线。
> - 删除、重命名、移动严格核验 Huawei `200 + File`，Move 使用 `addParentFolder/removeParentFolder`；响应丢失时按 fileId GET 核验后再结算。
> - 全局状态由单一聚合器按 revision 发布；前端拒绝旧快照，并区分当前同步失败、等待网络与传输历史失败。
>
> **1.2.0 变更（2026-07-10）**：
> - **网络守卫**（REQ-1）：新增 `net_guard` 模块——TCP 探测华为 API 域名 + 维护 Online/Offline 状态；断网/睡眠时保留同步请求，云端检查点未追平时不进入 planner
> - **数据丢失纵深防御**（BUG 4）：BFS 子树永久失败不再写 `complete=true`（第一道）；planner 启动恢复期对 `pending:+FAILED` 不再生成 Upload/Delete（第二道）；`validate_delete_from_local` 删前 API 复核（第三道）
> - **上传重试 + 断点续传**（BUG 2）：新增 `transfer_retry` 命令（非阻塞后台 spawn）；`upload_resume` 支持从 `ResumeSession.start_offset` 续传；DB schema v4 新增 `session_url` 列持久化续传令牌；传输面板失败项展示错误原因 + 重试按钮 + 实时进度刷新
> - **崩溃止血**（BUG 3）：`refresh_menu` 传输签名判等（无变化不重建菜单，降频）；panic hook 写 `last_crash.marker` + 启动检测日志
> - **日志导出加固**（REQ-2）：`logs_export` 加诊断日志 + UTF-8 容忍 + 前缀过滤；`cleanup_old_logs` 保留 30 天自动清理
> - **其他修复**：上传成功后补取 `editedTime` 防下载循环；`recentlyDeletedPaths` 改为 5 分钟过期 HashMap；目录删除只允许显式目录动作覆盖子孙，禁止把“全部直接文件删除”扩大为父目录删除

## 1. 项目概述

### 1.1 项目背景

华为云空间客户端目前尚未提供 macOS 版本。Mac 用户只能通过浏览器访问 cloud.huawei.com，无法与本地文件系统集成。

PetalLink 使用 Tauri 2.x 构建 macOS 桌面客户端，通过华为 Drive REST API 直连云盘，不依赖 HMS Core SDK（macOS 上不存在）。核心思路是把云盘内容映射到本地目录，在 Finder 中直接操作，本地与云端双向同步。

### 1.2 技术栈

| 层 | 技术 | 说明 |
|---|---|---|
| **后端** | Rust + Tauri 2.x | 业务逻辑、同步引擎、HTTP 客户端 |
| **前端** | Vue 3 + TypeScript + Vite + Pinia | Tauri 命令通信 |
| **数据库** | SQLite（rusqlite，bundled） | 同步状态、传输队列 |
| **HTTP** | reqwest（rustls-tls） | 华为 Drive REST API |
| **文件监听** | notify + notify-debouncer-full | macOS FSEvents，3s debounce |
| **安全存储** | `token.bin` + ChaCha20-Poly1305 AEAD | 密钥由本机 IOPlatformUUID 派生，文件权限 0600 |
| **加密/哈希** | sha2 / rand / base64 / hex | PKCE S256 / SHA256 |
| **单实例守护** | tauri-plugin-single-instance | 防双进程 FSEvents 冲突 |
| **系统托盘** | Tauri tray API + objc2-app-kit | NSStatusItem |
| **原生集成** | osascript / xattr / LaunchAgent plist | activationPolicy / Finder 灰标 / 开机自启 |
| **UI 设计** | Mate 组件库 + 自建设计令牌 | 主色 #0052D9 |
| **日志** | tracing + tracing-appender | stdout + 文件 + 环形缓冲 |

**华为 API 编码注意事项**（华为服务端与实际标准有偏差，以下处理均经过端到端验证）：

| 问题 | 根因 | 修复 |
|---|---|---|
| Scope `/` 被编码导致 invalid scope 1101 | scope 空格表示分隔，`/` 不编码 | scope 原始字符串拼接，空格用 `%20` |
| Authorization code 的 `+` 被当空格 → invalid code 1101 | form-urlencoded 标准行为 | 手工拼 form body，`percent_encoding` 精确编码 |
| Client 凭据注入 | 不能硬编码/提交 git | build.rs 自动读取 .env 中的 `HWCLOUD_CLIENT_ID` / `HWCLOUD_CLIENT_SECRET`，通过 `rustc-env` 注入编译期常量。缺失任一则 panic 阻断构建。运行时 dotenvy 兜底。两者均无硬编码默认值 |

### 2.8 文件编辑期间同步（三段式控制）

**第一阶段：变更捕获（Debounce）**
- FSEvents 监听到文件变化 → 启动 3s 去抖计时器
- 每次新变化重置计时器
- 计时器到期才进入下一阶段

**第二阶段：稳定性检查（Stability Check）**
- `sync/stability.rs`：mtime > 5s + size 稳定 3s + lsof 双重检查
- 任一不通过则重新等待
- lsof 白名单排除 Spotlight/mds/QuickLook 等系统只读进程

**第三阶段：上传前配额校验**
- 每次上传前调用 GET `/drive/v1/about` 获取 usedSpace 和 userCapacity
- 若文件大小 > remaining → 暂停上传，状态置为"失败：配额不足"

**特殊场景**

| 场景 | 处理 |
|---|---|
| 文件被持续编辑 > 5min | 标记 Editing，跳过自动同步；用户可手动触发上传 |
| 编辑期间 App 退出 | 下次启动重新触发 debounce + 稳定性检查 |
| 文件改名 | 检测到 rename 事件 → 同步改名到云端 |
| App 退出后用户修改文件 | `.hwcloud_syncstate` 快照启动时 diff 自动同步 |
| 复制文件（带 xattr） | `detect_renames` 检测到相同 fileId 时，检查旧文件是否仍存在 → 是则为复制（非改名），清除新文件旧 xattr，按全新文件上传 |

| 编号 | 功能 | 实现 |
|---|---|---|
| F-UPDATE-01 | 启动首次检查 | `App.vue` 启动 3s 后调用 `silentCheck()`（强制检查，不节流），命中更新仅更新侧边栏提示，不弹窗 |
| F-UPDATE-02 | 定时检查 | 每 1 小时由 `setInterval` 驱动 `periodicCheck()`，内部 `CHECK_INTERVAL_MS` 节流（60min）防止超频 |
| F-UPDATE-03 | 窗口聚焦检查 | 监听 `getCurrentWindow().onFocusChanged`，窗口获得焦点时调用 `checkOnFocus()`，内部 `FOCUS_THROTTLE_MS` 节流（10min）。覆盖从后台恢复、托盘/Dock 点击、单实例聚焦等所有「主窗口重新显示」路径 |
| F-UPDATE-04 | 手动检查 | 设置页「检查更新」按钮调用 `manualCheck()`（不节流），命中则弹出 `UpdateDialog` |
| F-UPDATE-05 | 侧边栏提示 | `Sidebar.vue`：检测到更新显示「新版本 vX.X.X」横幅 + 更新日志入口；下载进行中显示进度条 |
| F-UPDATE-06 | 下载安装 | `downloadAndInstall()` 下载更新包并展示进度；下载完成后 `waitForTransfers()` 轮询 `transfer_has_active` 等待同步队列空闲（最多 5min）再提示重启，避免更新重启打断进行中的同步任务 |
| F-UPDATE-07 | 资源清理 | `App.vue` `onUnmounted` 清理定时器与焦点监听句柄，防止泄漏 |

> **节流设计**：节流逻辑统一收敛在 `stores/updater.ts` 的 `throttledCheck()`，复用 `lastCheckTime` 时间戳。启动首次检查会初始化该时间戳，后续所有自动检查（定时 + 焦点）都遵守同一套节流规则。`manualCheck`（用户主动行为）不受节流限制。

### 6.4 占位文件 xattr 模型

为支持「按需下载」（Files-On-Demand-lite），云端文件在本地以**占位文件**形式存在：使用**真实文件名**（不加任何后缀）、**0 字节**空文件，仅作为目录结构占位与视觉标记，实际内容按需下载。占位文件的状态不依赖文件大小、文件名或文件内容，而是通过 macOS 扩展属性（xattr）独立追踪。

每个占位文件附带以下三个 xattr 键：

| xattr 键 | 取值 | 作用 |
|---|---|---|
| `com.hwcloud.fileId` | 云端文件 ID | 关联云端文件（创建时写入，运行时主要作记录用途） |
| `com.hwcloud.state` | `placeholder` 或 `downloaded` | **占位状态的唯一权威判据（source of truth）** |
| `com.hwcloud.size` | 真实云端大小（字节） | 记录云端真实大小（创建时写入，运行时无读取用途） |

其中 `com.hwcloud.state` 是判别占位文件的核心：取值为 `placeholder` 表示该文件尚未下载内容，取值为 `downloaded` 表示已下载真实内容。**xattr 是数据源头，文件大小、文件名、Finder 标签等均不参与占位状态的判定。**

此外，占位文件还会设置 **Finder 灰色标签**（label index 7）作为「未下载」的视觉反馈；下载完成后该标签会被清除。**Finder 标签仅作视觉提示，绝不用作判定占位状态的依据。**

**状态流转**：

- **创建占位**：云端发现本地缺失文件时，创建 0 字节文件，写入三个 xattr 并打灰标。若文件已存在但**无任何占位 xattr**，则视为用户文件，**绝不**转换为占位（防止误吞用户数据）。
- **下载完成**：xattr 状态由 `placeholder` 翻转为 `downloaded`，并清除灰标。
- **释放空间**：删除本地真实内容，重新创建占位文件并将状态置为 `placeholder`。
- **改后下载保护**：若占位文件被用户写入了内容（状态仍为 `placeholder` 但大小大于 0），下载云端新版本前会先将本地这份修改**改名备份**（形如「原名.local-时间戳.扩展名」），并清除备份文件的占位 xattr，使其被视为全新文件，随后将云端版本下载到原路径。

> 注：xattr 命名带 `hwcloud` 前缀，为项目历史命名遗留（项目现名 PetalLink），仅作键名，不影响功能。

## 7. 华为 Drive REST API 接口

### 7.1 认证端点

| 用途 | Method | Endpoint |
|---|---|---|
| 授权页 | GET | `https://oauth-login.cloud.huawei.com/oauth2/v3/authorize` |
| 换 Token / 刷新 | POST | `https://oauth-login.cloud.huawei.com/oauth2/v3/token` |
| 用户信息 (OIDC) | GET | `https://oauth-login.cloud.huawei.com/oauth2/v3/userinfo` |
| 显示名称 | POST | `https://account.cloud.huawei.com/rest.php?nsp_svc=GOpen.User.getInfo` |
| 手机号 | POST | `https://account.cloud.huawei.com/rest.php?nsp_svc=GOpen.User.getPhone` |

### 7.2 Drive API（全部 20 个接口已验证）

| 用途 | Method | Endpoint | 关键细节 |
|---|---|---|---|
| 用户配额 | GET | `/drive/v1/about` | 配额在 `storageQuota` 子对象，String 容忍 |
| 列举文件 | GET | `/drive/v1/files?fields=*&pageSize=100&queryParam=...` | `queryParam='id' in parentFolder` 语法 |
| 文件元数据 | GET | `/drive/v1/files/{id}` | |
| 创建文件夹 | POST | `/drive/v1/files` | mimeType 必填 `application/vnd.huawei-apps.folder` |
| 删除文件 | PATCH | `/drive/v1/files/{id}` | 请求体 `{"recycled":true}` 移入回收站（软删除）。DELETE 为永久删除，不使用 |
| 重命名/移动 | PATCH | `/drive/v1/files/{id}` | 中文名需 `\uXXXX` ASCII 转义 |
| 下载文件 | GET | `/drive/v1/files/{id}?form=content` | 流式 .tmp → rename 原子写 |
| 上传（小文件 ≤20MB） | POST | `/upload/drive/v1/files?uploadType=multipart` | multipart/related 格式 |
| 上传（大文件 >20MB） | POST → PUT | init 后始终使用 Location 会话 URL | 308 rangeList 只接受连续服务端确认范围；不确定分片先查询同一 session；401 原样重放一次 |
| 缩略图 | GET | `/drive/v1/thumbnails/{id}?form=content` | |
| 增量变更-取初始游标 | GET | `/drive/v1/changes/getStartCursor?fields=*` | cursor 必填；全量扫描必须在 BFS 前取得 startCursor |
| 增量变更-拉取变更 | GET | `/drive/v1/changes?fields=*&pageSize=100&includeDeleted=true&cursor=...` | nextCursor 翻页，末页 newStartCursor 提交；`deleted=true` 支持无 File tombstone；本地 trashDone/recycled 兼容 |

### 7.3 中文文件名编码

华为 Drive API 服务端 JSON 解析器**不接受 UTF-8 多字节字符**直接出现在 JSON 值中（即使 Content-Type 声明 charset=utf-8），需将所有 > 0x7F 码点转义为 `\uXXXX`（`drive/ascii_json.rs`，含 UTF-16 代理对处理）。

### 7.4 防振荡守卫

华为 PATCH `recycled:true` 后 LIST 仍可能返回旧数据（最终一致性），`sync/engine.rs` 的 `recentlyDeletedPaths` 追踪刚通知云端删除的路径，拦截 stale 数据导致的误判动作（createFolder / deleteFromLocal），防止无限振荡。

## 9. 关键设计决策

| 决策项 | 选择 | 理由 |
|---|---|---|
| **OAuth Scope** | `openid profile drive`（全盘访问） | `drive.file` 只能看本应用创建/打开的文件，网页/其他客户端上传的看不到 |
| **OAuth 回调** | 127.0.0.1:9999 loopback HTTP | AGC 接受；不用 URL Scheme |
| **云端变更检测** | 定时增量（changes API）+ 全量 BFS 兜底 | 华为无 webhook；纯定时全量轮询对大网盘（17K+ 文件）是灾难。改为：定时器（默认 15min）优先走 `/drive/v1/changes` 增量（cursor 持久化，只拉变更），失败/cursor 过期/连续达阈值（20 次）自动回退全量 BFS。缓存使非首次启动从 4min 降至 200ms |
| **本地变更处理** | FSEvents + 3s debounce + 稳定性检查 + 配额校验 | 完整三段式控制；纯事件驱动，无定时轮询 |
| **首次同步** | 手动引导（SyncSetupBanner → 选空目录 → 点「同步索引」） | 避免覆盖用户本地已有内容 |
| **默认挂载目录** | 空（不设默认） | 用户未配置时不上传/同步任何内容 |
| **退出后目录** | 保持可写，启动时按 `.hwcloud_syncstate` 快照自动 diff | 最自然体验 |
| **冲突策略** | 60s 容忍窗口 + 副本去重 + 目录保护 | 本地时钟偏差免疫；云端删目录时保护本地修改 |
| **沙盒** | 关闭（`app-sandbox = false`） | 双向同步需读写用户挂载目录；自用侧载工具 |
| **Token 存储** | `token.bin` 加密文件（chmod 600） | dev/release 通过 bundle id 隔离目录；不依赖 Keychain 签名状态 |
| **中文文件名** | `asciiJsonEncode`（>0x7F → \uXXXX） | 华为 Java JSON 解析器不接受 UTF-8 直接传输 |
| **内部文件隔离** | `.hwcloud_` 前缀全局硬编码过滤 | 内部缓存/快照文件绝不能参与同步 |
| **syncFolderRecursive 争用** | `_syncing` 互斥锁 | 并发执行中跳过，防重复上传/误判 |
| **云端树索引完整性** | `complete` 标记 + 哨兵 + 原子写 | 防止半截缓存被用 → planner 全量误判 |
| **FSEvents 历史回放** | 8s warmup 窗口丢弃 | macOS 新 watcher 注册时会回放进程生命周期内的历史事件；8s 窗口覆盖多个 debounce 周期，确保 BFS 建目录/占位过程中的事件不被误触发 |
| **activationPolicy** | osascript 切换（regular ↔ accessory）+ `--hidden` 启动来源检测 | 备用 objc2-app-kit 方案需 feature gate |
| **Token 隔离** | dev bundle id 带 `-dev` | dev 与正式版使用不同 Application Support 目录，避免 token 和同步状态混用 |
| **后台运行** | Tauri 关窗/退出事件拦截 + 托盘「退出」唯一真退出路径 | 单进程双层：UI 层隐藏/显示 + 后台引擎继续运行 |
| **占位状态判据** | 以 `com.hwcloud.state` xattr 为唯一权威判据 | 文件大小（0 字节）、文件名、Finder 灰标等都只是占位文件的伴随特征，单独不足以判定；尤其要避免把用户的 0 字节空文件（如 `.gitkeep`、空配置）误判为占位符。xattr 独立于文件内容，扫描/下载/释放/删除等各路径口径统一。仅 macOS 支持扩展属性 |
| **应用内自动更新签名** | 免费 Tauri 更新签名（`tauri signer generate` 生成的 Ed25519 密钥对），不购买 Apple 开发者账号 | Tauri 更新器强制校验签名且不可关闭；自动更新走 `.app.tar.gz` + `.sig` 校验链路，**不经过 Gatekeeper**，因此无需 Apple 代码签名。公钥写 `tauri.conf.json` 的 `plugins.updater.pubkey`，私钥 `TAURI_SIGNING_PRIVATE_KEY` 存为 GitHub Secret。Apple 签名（$99/年）仅影响首次 DMG 双击是否被拦截，与自动更新无关。`createUpdaterArtifacts: true` 让 Tauri 构建期自动生成 `.app.tar.gz` + `.sig` |

## 10. 已知限制与风险

| 项目 | 说明 |
|---|---|
| **activationPolicy** | 当前用 osascript 实现，正式版可改 objc2-app-kit 直接调 NSApp |
| **签名** | DMG 未签名，分发需 Apple Developer ID 签名 + 公证 |
| **侧边栏深层递归** | 当前支持递归展开，大量目录时可优化为虚拟滚动 |
| **dock 图标（dev 模式）** | `cargo tauri dev` 跑裸二进制非 .app 包，Dock 图标可能显示直角；release .app 包图标正常 |
| **会话内换账号** | 需清 DB/缓存/重置目录配置 + 原地重启引擎，不再保留旧状态 |
| **bundle id 大小写** | 保持现有 bundle id | Application Support 路径与 bundle id 绑定，变更会产生新的数据目录 |
| **华为 API 最终一致性** | DELETE 后 LIST 仍可返回旧数据，`recentlyDeletedPaths` 防振荡守卫（5 分钟过期） |
| **配额字段 String 容忍** | `DriveAbout` 的 `usedSpace` / `userCapacity` 可能是 String 或 Number，已做兼容解析 |
| **大文件断点续传** | session URL 和本地提示 offset 会持久化，但恢复时必须查询同一 session 的服务端连续 range；session 过期或查询不确定时保持 RemoteAmbiguous，先核验目标，绝不盲建第二个文件。 |
| **网络断开 / 睡眠期间** | Running 的可恢复失败转 WaitingForNetwork，退避错误转 BackingOff，写入响应丢失转 VerifyingRemote；探测/请求边沿唤醒统一恢复 owner。下载 `.tmp` 在可恢复错误下保留，永久失败才清理。 |
| **changes cursor 过期** | 过期或任何分页/merge 协议错误都保留旧 tree+cursor checkpoint 并回退“startCursor before BFS + replay”全量流程；成功提交前 destructive trust 为 false。 |
| **增量同步对改名/移动的局限** | `merge_changes_into_cloud_tree` 按 fileId 反查 rel_path，对"已知 id 但路径已变"（改名/移动）会更新到旧 rel_path，与真实路径不一致。新增不动点循环反向解析：利用 `Change.file.parent_folder` 解析新文件/恢复文件的完整路径。仍无法解析的（如从回收站恢复后 parent_folder 缺失）→ skip → 检测到全 skip 后自动回退全量 BFS。 |
| **占位 xattr 无完整性校验** | 占位状态以 `com.hwcloud.state` xattr 为绝对依据，但不做签名、不与 DB 交叉核对、不校验内容哈希。若用户手动篡改 xattr，会直接改变系统对该文件状态的认知：① 删除占位文件的 state 且无 DB 记录时，0 字节空文件可能被当作用户新建文件误上传到云端；② 将已下载大文件的 state 改回 placeholder 时，若云端有更新触发下载，真实内容会被改名备份后由云端版本覆盖；③ 前端状态显示（依赖磁盘 xattr）与同步引擎决策（更依赖 DB 与 mtime/size）可能不一致。此外 DB 的 `status` 字段与磁盘 xattr 为双轨制，出现分歧时不做仲裁。 |
