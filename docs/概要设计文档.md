# PetalLink 概要设计文档

> 版本：1.0.0 · 2026-06-26
> 技术栈：Tauri 2.x · Rust 后端 · Vue3/TS 前端

---

## 1. 项目概述

### 1.1 项目背景

华为云空间客户端目前尚未提供 macOS 版本。Mac 用户只能通过浏览器访问 cloud.huawei.com，无法与本地文件系统集成。

PetalLink 使用 Tauri 2.x 构建 macOS 桌面客户端，通过华为 Drive REST API 直连云盘，不依赖 HMS Core SDK（macOS 上不存在）。核心思路是把云盘内容映射到本地目录，在 Finder 中直接操作，本地与云端双向同步。

### 1.2 技术栈

| 层 | 技术 | 说明 |
|---|---|---|
| **后端** | Rust + Tauri 2.x | 业务逻辑、同步引擎、HTTP 客户端 |
| **前端** | Vue 3 + TypeScript + Vite + Pinia | Tauri 命令通信 |
| **数据库** | SQLite（rusqlite，bundled） | 同步状态、传输队列 |
| **HTTP** | reqwest（rustls-tls） | 华为 Drive REST API |
| **文件监听** | notify + notify-debouncer-full | macOS FSEvents，3s debounce |
| **安全存储** | keyring（macOS Keychain） | Keychain 优先，文件降级 |
| **加密/哈希** | sha2 / rand / base64 / hex | PKCE S256 / SHA256 |
| **单实例守护** | tauri-plugin-single-instance | 防双进程 FSEvents 冲突 |
| **系统托盘** | Tauri tray API + objc2-app-kit | NSStatusItem |
| **原生集成** | osascript / xattr / LaunchAgent plist | activationPolicy / Finder 灰标 / 开机自启 |
| **UI 设计** | Mate 组件库 + 自建设计令牌 | 主色 #0052D9 |
| **日志** | tracing + tracing-appender | stdout + 文件 + 环形缓冲 |

## 2. 功能需求

### 2.1 授权与认证（F-AUTH）

| 编号 | 功能 | 实现 |
|---|---|---|
| F-AUTH-01 | OAuth 2.0 + PKCE 登录 | Rust `auth/service.rs` 编排完整流程（authorize → code → token）。scope: `openid profile https://www.huawei.com/auth/drive`（全盘访问，不能用 drive.file，否则看不到其他客户端上传的文件） |
| F-AUTH-02 | 本地 HTTP 回调服务器 | 127.0.0.1:9999 loopback（`auth/oauth_server.rs`，5min 超时），绑定 127.0.0.1 不监听 0.0.0.0 |
| F-AUTH-03 | Token 持久化 | `keyring` crate → macOS Keychain；Keychain 不可用时自动降级到文件存储（chmod 600） |
| F-AUTH-04 | Token 自动刷新 | `auth/token_refresher.rs`：expiry 前 60s 主动刷新 + HTTP 401 触发强制刷新；`AsyncMutex` 并发去重 |
| F-AUTH-05 | 用户信息聚合 | `auth/user_info_api.rs`：三端点并行合并（OIDC userinfo < displayName < phone），单端点失败不阻断 |
| F-AUTH-06 | 退出登录 | 清空 Keychain token + DB 记录 + cloudtree/syncstate 缓存 + 重置 config 挂载字段 |
| F-AUTH-07 | 登录态守卫 | setup 加载 config 前先检查 token；未登录但旧配置残留 → 自动清理孤儿状态 |

**华为 API 编码注意事项**（华为服务端与实际标准有偏差，以下处理均经过端到端验证）：

| 问题 | 根因 | 修复 |
|---|---|---|
| Scope `/` 被编码导致 invalid scope 1101 | scope 空格表示分隔，`/` 不编码 | scope 原始字符串拼接，空格用 `%20` |
| Authorization code 的 `+` 被当空格 → invalid code 1101 | form-urlencoded 标准行为 | 手工拼 form body，`percent_encoding` 精确编码 |
| Client 凭据注入 | 不能硬编码/提交 git | build.rs 自动读取 .env 中的 `HWCLOUD_CLIENT_ID` / `HWCLOUD_CLIENT_SECRET`，通过 `rustc-env` 注入编译期常量。缺失任一则 panic 阻断构建。运行时 dotenvy 兜底。两者均无硬编码默认值 |

### 2.4 挂载同步（F-MOUNT）

| 编号 | 功能 | 实现 |
|---|---|---|
| F-MOUNT-01 | 挂载目录选择 | 首次启动引导用户选择空目录（`mount_configured` 标志区分"默认值"与"用户已确认"） |

| F-MOUNT-03 | 双向同步引擎 | `sync/engine.rs`：3-way diff（本地 vs 云端 vs DB）→ 决策表 → 执行（13 种情况 + startup-resume 特殊分支）；`_syncing` 互斥锁防竞态 |
| F-MOUNT-04 | 本地变更监听 | `mount/local_watcher.rs`：notify crate（macOS FSEvents）+ 3s debounce 计时器；FSEvents 历史回放防护（8s warmup 窗口）；纯事件驱动，无定时轮询 |
| F-MOUNT-05 | 云端变更同步 | `sync/cloud_tree.rs`：BFS 8 并发拉取全量目录树 + 持久化到 `.hwcloud_cloudtree.json`（含 `complete` 完整性标记）；非首次启动从磁盘缓存秒级加载；用户点「同步索引」手动触发全量拉取 |
| F-MOUNT-06 | 同步状态存储 | SQLite `sync_items` 表：fileId、相对路径、sha256、mtime、status；DB 文件为 `petal_link.db`（schemaVersion=3） |
| F-MOUNT-07 | 启动时首次同步 | 全量比对后增量同步（startup-resume 模式） |
| F-MOUNT-08 | 后台运行 | 关窗拦截：CloseRequested → `api.prevent_close()` + 隐藏窗口 + `set_accessory()`（Dock 图标消失）；ExitRequested（Cmd+Q/Dock Quit）→ `api.prevent_exit()` + 隐藏 + accessory；仅托盘「退出 PetalLink」置 `mark_real_quit()` 真正退出 |
| F-MOUNT-09 | 变更去抖 debounce | 默认 3 秒（notify-debouncer-full），期间持续变化则重置计时器 |
| F-MOUNT-10 | 文件稳定性检查 | `sync/stability.rs`：①mtime 距今 > 5s ②size 连续 3s 不变 ③lsof 无进程以写模式打开（双重检查 + 只读系统进程白名单） |
| F-MOUNT-11 | 持续编辑暂停 | lsof 检测到进程占用或 size 持续增长 > 5min → 标记 Editing，跳过自动同步 |
| F-MOUNT-12 | 并发数可配置 | 默认 6，范围 1-20 |
| F-MOUNT-13 | 首次同步手动引导 | 登录后不自动同步，顶部 SyncSetupBanner 引导选空目录 → 「同步索引」按钮 → 手动触发后转自动 |
| F-MOUNT-14 | 退出后可写 + 启动 diff | App 退出后同步目录保持可写；`.hwcloud_syncstate` JSON 快照记录上次状态；下次启动自动 diff |
| F-MOUNT-15 | Agent 模式 + 开机自启 | `--hidden` 参数检测启动来源（`activation.rs:is_launched_manually`）；`launch_at_login.rs`：写 `~/Library/LaunchAgents/<BUNDLE_ID>.plist`，ProgramArguments 含 `--hidden` |
| F-MOUNT-16 | 云端树持久化缓存 | `sync/cloud_tree.rs`：BFS 完成后原子写 `.hwcloud_cloudtree.json`（含 `complete` 标记）；索引完整性守卫——`complete:false` 哨兵防止半截缓存被误用；非首次启动从磁盘秒级加载 |
| F-MOUNT-17 | 内部文件隔离 | `.hwcloud_` 前缀全局硬编码过滤（`mount/skip.rs`：`mount/manager.rs` / `mount/local_watcher.rs` / `sync/engine.rs` / `sync/cloud_tree.rs` / DB 记录加载 全部排除） |

### 2.8 文件编辑期间同步（三段式控制）

**第一阶段：变更捕获（Debounce）**
- FSEvents 监听到文件变化 → 启动 3s 去抖计时器
- 每次新变化重置计时器
- 计时器到期才进入下一阶段

**第二阶段：稳定性检查（Stability Check）**
- `sync/stability.rs`：mtime > 5s + size 稳定 3s + lsof 双重检查
- 任一不通过则重新等待
- lsof 白名单排除 Spotlight/mds/QuickLook 等系统只读进程

**第三阶段：上传前配额校验**
- 每次上传前调用 GET `/drive/v1/about` 获取 usedSpace 和 userCapacity
- 若文件大小 > remaining → 暂停上传，状态置为"失败：配额不足"

**特殊场景**

| 场景 | 处理 |
|---|---|
| 文件被持续编辑 > 5min | 标记 Editing，跳过自动同步；用户可手动触发上传 |
| 编辑期间 App 退出 | 下次启动重新触发 debounce + 稳定性检查 |
| 文件改名 | 检测到 rename 事件 → 同步改名到云端 |
| App 退出后用户修改文件 | `.hwcloud_syncstate` 快照启动时 diff 自动同步 |
| 复制文件（带 xattr） | `detect_renames` 检测到相同 fileId 时，检查旧文件是否仍存在 → 是则为复制（非改名），清除新文件旧 xattr，按全新文件上传 |

## 3. 非功能需求

### 3.1 性能

| 指标 | 要求 | 实现 |
|---|---|---|
| UI 响应 | 任意操作响应时间 < 200ms | Rust 后端异步处理，主线程不阻塞 |
| 并发传输 | 默认 6 个并发（可配置 1-20） | `sync/executor.rs`：tokio task 并发 |
| 同步扫描 | 1000 文件目录扫描 < 10s | BFS 8 并发 + 云端树磁盘缓存 |
| 内存占用 | 空闲 < 200MB，传输高峰 < 500MB | Rust 编译优化（lto + opt-level=s） |
| 启动速度 | 非首次 < 1s | 云端树缓存 ~200ms（17K 文件） |

### 3.2 安全性

| 指标 | 要求 |
|---|---|
| Token 存储 | macOS Keychain（`keyring` crate）；Keychain 不可用时降级文件存储（chmod 600） |
| Client 凭据 | build.rs 自动读取 .env 的 `HWCLOUD_CLIENT_ID` / `HWCLOUD_CLIENT_SECRET` 编译期注入，缺失则 panic 阻断构建。不硬编码，不提交 git |
| 网络通信 | 全程 HTTPS（华为 API 强制 TLS） |
| 本地回调 | 仅绑定 127.0.0.1:9999，不监听 0.0.0.0 |
| 日志 | 不打印完整 token / secret |

### 4.2 后端模块划分

| 模块 | 路径 | 职责 |
|---|---|---|
| `auth/` | `src/auth/` | OAuth 流程、PKCE、Token 存储/刷新、用户信息聚合 |
| `core/` | `src/core/` | config（配置模型）、config_store（JSON 持久化）、env_loader（.env 解析）、logging（tracing 三层输出）、cache_paths（路径管理） |
| `drive/` | `src/drive/` | REST API 封装：files_api、upload_api、download_api、thumbnail_api、about_api；ascii_json（中文 \uXXXX 转义）；models（DriveFile / DriveAbout） |
| `mount/` | `src/mount/` | manager（镜像目录/占位），local_watcher（FSEvents + debounce），file_hasher（SHA256），skip（跳过规则） |

### 7.3 中文文件名编码

华为 Drive API 服务端 JSON 解析器**不接受 UTF-8 多字节字符**直接出现在 JSON 值中（即使 Content-Type 声明 charset=utf-8），需将所有 > 0x7F 码点转义为 `\uXXXX`（`drive/ascii_json.rs`，含 UTF-16 代理对处理）。

### 7.4 防振荡守卫

华为 DELETE 返回 204 后 LIST 仍可能返回旧数据（最终一致性），`sync/engine.rs` 的 `recentlyDeletedPaths` 追踪刚通知云端删除的路径，拦截 stale 数据导致的误判动作（createFolder / deleteFromLocal），防止无限振荡。

## 9. 关键设计决策

| 决策项 | 选择 | 理由 |
|---|---|---|
| **OAuth Scope** | `openid profile drive`（全盘访问） | `drive.file` 只能看本应用创建/打开的文件，网页/其他客户端上传的看不到 |
| **OAuth 回调** | 127.0.0.1:9999 loopback HTTP | AGC 接受；不用 URL Scheme |
| **云端变更检测** | BFS 全量拉取 + 磁盘缓存（`.hwcloud_cloudtree.json`） | 华为无 webhook；定时轮询对大网盘（17K+ 文件）是灾难；缓存使非首次启动从 4min 降至 200ms |
| **本地变更处理** | FSEvents + 3s debounce + 稳定性检查 + 配额校验 | 完整三段式控制；纯事件驱动，无定时轮询 |
| **首次同步** | 手动引导（SyncSetupBanner → 选空目录 → 点「同步索引」） | 避免覆盖用户本地已有内容 |
| **默认挂载目录** | 空（不设默认） | 用户未配置时不上传/同步任何内容 |
| **退出后目录** | 保持可写，启动时按 `.hwcloud_syncstate` 快照自动 diff | 最自然体验 |
| **冲突策略** | 60s 容忍窗口 + 副本去重 + 目录保护 | 本地时钟偏差免疫；云端删目录时保护本地修改 |
| **沙盒** | 关闭（`app-sandbox = false`） | 双向同步需读写用户挂载目录；自用侧载工具 |
| **Token 存储** | Keychain 优先，文件降级（chmod 600） | 自适应：有签名用 Keychain，无签名降级 |
| **中文文件名** | `asciiJsonEncode`（>0x7F → \uXXXX） | 华为 Java JSON 解析器不接受 UTF-8 直接传输 |
| **内部文件隔离** | `.hwcloud_` 前缀全局硬编码过滤 | 内部缓存/快照文件绝不能参与同步 |
| **syncFolderRecursive 争用** | `_syncing` 互斥锁 | 并发执行中跳过，防重复上传/误判 |
| **云端树索引完整性** | `complete` 标记 + 哨兵 + 原子写 | 防止半截缓存被用 → planner 全量误判 |
| **FSEvents 历史回放** | 8s warmup 窗口丢弃 | macOS 新 watcher 注册时会回放进程生命周期内的历史事件；8s 窗口覆盖多个 debounce 周期，确保 BFS 建目录/占位过程中的事件不被误触发 |
| **activationPolicy** | osascript 切换（regular ↔ accessory）+ `--hidden` 启动来源检测 | 备用 objc2-app-kit 方案需 feature gate |
| **Keychain 前缀** | `hwcloud.` | 与旧版凭据前缀保持一致，避免切换 bundle id 后丢失已授权的 token |
| **后台运行** | Tauri 关窗/退出事件拦截 + 托盘「退出」唯一真退出路径 | 单进程双层：UI 层隐藏/显示 + 后台引擎继续运行 |

## 10. 已知限制与风险

| 项目 | 说明 |
|---|---|
| **activationPolicy** | 当前用 osascript 实现，正式版可改 objc2-app-kit 直接调 NSApp |
| **签名** | DMG 未签名，分发需 Apple Developer ID 签名 + 公证 |
| **侧边栏深层递归** | 当前支持递归展开，大量目录时可优化为虚拟滚动 |
| **dock 图标（dev 模式）** | `cargo tauri dev` 跑裸二进制非 .app 包，Dock 图标可能显示直角；release .app 包图标正常 |
| **会话内换账号** | 需清 DB/缓存/重置目录配置 + 原地重启引擎，不再保留旧状态 |
| **bundle id 大小写** | 含大写字母，Keychain 凭据不跨新旧 bundle id 通用 |
| **华为 API 最终一致性** | DELETE 后 LIST 仍可返回旧数据，`recentlyDeletedPaths` 防振荡守卫 |
| **配额字段 String 容忍** | `DriveAbout` 的 `usedSpace` / `userCapacity` 可能是 String 或 Number，已做兼容解析 |

## 11. 构建、测试与打包

开发、测试和生产打包的完整命令见 [README.md](../README.md) 中的对应章节。以下仅列出概要：

**环境依赖**：Rust 1.77+、Node.js 20+、macOS 12+、Xcode CLI

```bash
# 开发
cargo tauri dev                           # Rust 后端 + Vite 前端 HMR

# 测试
cargo test --lib                          # 164 个 Rust 单元测试
cargo test                                # 全部（含 12 个 wiremock 集成测试）
cargo clippy -- -D warnings               # Rust 代码质量
cd app && npm run type-check              # Vue TS 类型检查
cd app && npm run test                    # Vitest 前端测试

# 发布（build.rs 自动从 .env 注入 HWCLOUD_CLIENT_ID / HWCLOUD_CLIENT_SECRET，无需手动设环境变量）
cargo tauri build
# 产物: target/release/bundle/macos/PetalLink.app
#       target/release/bundle/dmg/PetalLink_1.0.0_aarch64.dmg
```

**打包配置**：Release profile 启用 `lto=true`、`opt-level=s`、`strip=true`、`panic=abort`。`build.rs` 自动从 `assets/` 同步图标到 `icons/` 并生成 `icon.icns`。

**分发签名**（分发给他人时必需 Apple Developer ID 签名 + 公证），自用可直接 `xattr -cr` 绕过 Gatekeeper。
