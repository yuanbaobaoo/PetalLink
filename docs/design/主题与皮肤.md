# 主题与皮肤

> **来源**：petal-link-cmp `docs/`（2026-07 纳入 Flutter 项目）。术语映射：CMP ViewModel ≈ Flutter GetX Controller；Room ≈ sqflite；StateFlow ≈ Rx/Stream。文件身份识别以 **inode 方案**为准（见 `10-基于inode的文件身份识别方案.md`），文中如涉及旧 xattr fileId 描述均为历史背景。


Flutter 没有 Sass variables；本项目用强类型的 `MateSkin`（对标 CMP `PetalSkin`）承担同样职责。唯一入口是：

- `lib/app/theme/mate_tokens.dart`：默认字体、默认尺寸和原始色板的实际取值（对标 CMP DesignTokens.kt）
- `lib/app/theme/mate_typography.dart`：逐项字体字段定义（对标 CMP TypographyTokens.kt）
- `lib/app/theme/mate_metrics.dart`：Skin 尺寸入口和基础尺寸字段定义（对标 CMP CoreMetrics.kt）
- `lib/app/theme/mate_skin.dart`：明暗皮肤实例（`MateSkin.light` / `MateSkin.dark`）
- `lib/app/theme/mate_metrics.dart`：通用组件尺寸（对标 CMP ComponentMetrics.kt）
- `lib/app/theme/mate_metrics.dart`：登录、日志、主页面和浮层尺寸（对标 CMP PageMetrics.kt）
- `lib/app/theme/mate_metrics.dart`：设置页 / 文件列表 / 侧边栏 / 传输 / 更新界面等分域尺寸
- `lib/app/theme/mate_semantic_colors.dart`：浅色、深色和逐项语义颜色（对标 CMP Theme.kt）
- `lib/app/theme/mate_skin.dart`：皮肤数据结构和 UI 读取入口（对标 CMP PetalSkin.kt）

## 调整单个 UI 职责

变量按 UI 职责拆分，不按字号档位复用。下面两个文字即使都是 `14.sp`，也必须分别修改：

```kotlin
val MY_TYPOGRAPHY = DesignTokens.TYPOGRAPHY.copy(
    button = DesignTokens.TYPOGRAPHY.button.copy(
        primaryLabel = DesignTokens.TYPOGRAPHY.button.primaryLabel.copy(fontSize = 15.sp),
    ),
    menu = DesignTokens.TYPOGRAPHY.menu.copy(
        itemLabel = DesignTokens.TYPOGRAPHY.menu.itemLabel.copy(fontSize = 13.sp),
    ),
)
```

不要新增 `font14`、`bodySize` 之类会被多个职责共享的变量。新变量应说明它控制的具体位置，例如 `button.primaryLabel`、`menu.itemLabel`、`fileList.rowFileName`。

## 新增皮肤

默认皮肤是不可变数据，可通过 `copy` 只覆盖需要变化的部分：

```kotlin
val COMPACT_SKIN = DEFAULT_PETAL_SKIN.copy(
    name = "compact",
    typography = DesignTokens.TYPOGRAPHY.copy(
        button = DesignTokens.TYPOGRAPHY.button.copy(
            primaryLabel = DesignTokens.TYPOGRAPHY.button.primaryLabel.copy(fontSize = 13.sp),
        ),
        menu = DesignTokens.TYPOGRAPHY.menu.copy(
            itemLabel = DesignTokens.TYPOGRAPHY.menu.itemLabel.copy(fontSize = 12.sp),
        ),
    ),
    metrics = DesignTokens.METRICS.copy(
        button = DesignTokens.METRICS.button.copy(primaryHeight = 32.dp),
        menu = DesignTokens.METRICS.menu.copy(itemHeight = 32.dp),
    ),
)
```

在应用根节点启用：

```kotlin
PetalLinkTheme(skin = COMPACT_SKIN) {
    App()
}
```

所有页面只读取 `PetalTheme.typography`、`PetalTheme.metrics` 和 `PetalTheme.colors`，不要在页面中直接写字号或业务颜色。

## 颜色写法

Flutter 的 `Color` 构造函数接收数值，不直接接收 CSS 的 `"#AABBCC"` 字符串。项目中的：

```kotlin
Color(0xFF173A31)
```

是十六进制 ARGB，不是八进制：`0x` 表示十六进制，前两位 `FF` 是完全不透明的 Alpha，后六位 `173A31` 等同 CSS 的 `#173A31`。它只出现在主题定义中；页面通过有用途说明的 `PetalTheme.colors` 字段读取颜色。

不建议为了写 `"#173A31"` 增加字符串解析器：这会把编译期可检查的数值变成运行时解析，也不符合 Flutter 原生 API。需要从配置文件动态读取颜色时，再单独提供字符串解析入口。
