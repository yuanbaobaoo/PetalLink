# 11 · 当前 CMP 实现审计

> 审计日期：2026-07-16  
> 对照基线：`docs/plan`、`docs/reference`、原仓库 `/Users/Shared/codes/personal/petal-link` 及当前 CMP 源码。

## 1. 总体判断

当前实现属于“核心纯逻辑原型 + Compose UI 壳”，尚未形成可供真实用户使用的纵向闭环。

- 用户可用闭环估算：15%–25%
- 纯逻辑模块估算：40%–50%
- `:shared:jvmTest`：通过，当前生成 171 个测试用例结果
- 尚未验证：真实 OAuth、首次数据库建库、真实 Drive 合同、FSEvents、xattr、真实传输、后台生命周期、DMG 运行

在完成 P0 安全阻断项前，不应连接真实账号执行云端写操作，也不应对真实同步目录调用释放空间。

## 2. 已有基础

### 2.1 已实现并具备测试的纯逻辑

- PKCE URL 编码和 token 二进制序列化
- 非 ASCII 字符 `\uXXXX` 转换函数
- 308 `rangeList` 连续性校验
- Changes 删除信号和 cursor 部分规则
- 配置基础校验
- 网络状态防抖纯状态机
- 九态枚举与迁移矩阵
- planner 的部分三方 diff 分支
- 冲突副本命名的部分逻辑
- 防振荡、ActivityTracker、CycleCoordinator 的部分纯逻辑
- SHA-256、skip filter、稳定性判定纯函数
- SQLDelight repository 接口和 inode/free-up 表骨架
- Compose 主题、部分 Mate 组件和文件列表组件

### 2.2 已存在但未接入主流程的模块

- `CommandService`
- `AuthService`、`DriveClient` 和各 API 类
- `SyncEngine`、`TaskRunner`、`SyncExecutor`
- `StatusAggregator`
- Auth/Sync/Transfer/Updater ViewModel
- 平台接口定义

“代码存在”不等于“功能完成”。以上对象均未从桌面应用入口形成完整装配和生命周期管理。

## 3. P0 阻断项

### 3.1 应用入口是静态演示

`Main.kt` 当前使用硬编码登录状态和示例文件，没有创建数据库、CommandService、NetGuard 或 SyncEngine。关闭窗口直接退出应用。

影响：所有业务实现均无法从 UI 到达，后台同步和状态流不存在。

### 3.2 OAuth 登录不可完成

当前登录命令没有生成 PKCE/state、没有打开浏览器、没有校验回调 state，换 token 时 verifier 为 null；取消登录为空实现，用户信息固定为空。

logout 只删除 token，没有清理 DB、云树、同步快照和挂载配置。

### 3.3 Drive DTO 与协议不一致

当前 `DriveFile` 主要使用 snake_case `SerialName`，而华为响应使用 `fileName`、`mimeType`、`parentFolder`、`editedTime`。文件夹 MIME 判断也缺少 `application/` 前缀。

影响：列表、BFS、planner、Changes、上传完成核验均可能得到空字段或错误类型。

### 3.4 Files API 存在破坏性错误

- 列目录丢弃实际 parentId，发送固定字面量 `'id' in parentFolder`
- search 退化为本地过滤
- move 未实现
- delete 没有核验响应 File 的 id/recycled
- create/update 的中文 JSON 组装可能产生双重转义
- 没有严格 schema 校验和不确定写远端收敛

### 3.5 上传下载未形成协议闭环

- 命令层下载只读元数据，不写目标文件
- 按需下载同样没有下载内容
- 上传命令只走小文件并一次性读入内存
- 没有分片 PUT、断点恢复、最终状态轮询和 Update 上传
- 下载执行没有 Range 恢复、sidecar、416 单次回退、流式写入、fsync 和二次元数据核验

### 3.6 数据库首次启动与迁移不完整

JDBC driver 创建后没有明确执行 `PetalLinkDatabase.Schema.create(driver)`，也没有 v2→v6 migration。

当前 `transfer_queue` 缺少原终态 schema 的 operation、error_kind、next_retry_at、source snapshot、resume 上下文和 remote result 等关键字段。

### 3.7 同步引擎无法通过信任门

全量 BFS 返回的 cursor 固定为 null，而可信云树要求 cursor 非空。增量刷新目前只是再次执行全量刷新，且没有 checkpoint 持久化。

`CommandService.create()` 传入的 `syncPlan` 固定为 null，因此手动刷新和失败重试必然返回 engine not started。

### 3.8 本地挂载和监听未实现

尚缺：递归扫描、inode 读取、move detection、xattr、Finder 灰标、FSEvents、debounce/warmup、generation、lsof、PlaceholderManager 的 JVM 实现。

### 3.9 释放空间存在数据丢失风险

当前单项和批量释放空间直接调用 `Files.deleteIfExists`，没有 13 步 TOCTOU 核验、staging rename、DB 事务、远端复核和崩溃恢复。

### 3.10 macOS 生命周期为空壳

托盘、单实例、activationPolicy、LaunchAgent、优雅关闭和系统关机识别只有接口或空实现。JNA FSEvents/xattr 方法不执行实际操作。

## 4. P1 功能缺口

- 49 个原命令缺少 `driveListAll`，已有命令中多项只是占位
- transfer active 只检查 Running，没有包括 Pending
- clear completed/failed/finished 语义错误或为空操作
- sync items by folder 退化成精确路径查询
- StatusAggregator 只发布五态枚举，没有完整计数、revision、failed items 和 runtime phase
- SyncViewModel 不拒绝比当前更旧的 revision
- 缺少 fileBrowser ViewModel 和真实目录树联动
- 缺少完整多选、批量操作、移动、重命名、属性和右键菜单
- 缺少日志滚动文件、30 天清理和应用级共享 ring buffer
- ConfigStore 吞掉保存错误，配置字段缺少 mountConfigured、skipPatterns、sort 等
- 更新器只有 UI phase 枚举，没有检查、下载、签名和安装实现

## 5. 文档与构建缺口

- 旧 README 和 Compose Multiplatform (CMP) 路线已移除，当前文档以 CMP JVM 为唯一目标
- Gradle 仅配置基础 DMG/MSI，缺少图标、签名、公证和发布 CI
- 版本号固定在构建脚本，尚无统一版本来源
- 原 Tauri release 规则仍需在实施发布阶段替换为 CMP Desktop 规则

## 6. 完成判定

只有满足以下条件，项目才可从“原型”进入“可用 Alpha”：

1. 应用启动完成数据库、配置、日志、网络、服务和引擎装配。
2. 真实 OAuth 可以登录、恢复、刷新、取消和退出。
3. Mock HTTP 合同覆盖全部华为 API 怪癖和写后核验。
4. 临时目录中能完成一次本地新增上传、云端新增占位和按需下载。
5. 重启后能恢复云树 checkpoint 和中断传输，不重复创建远端文件。
6. FSEvents、xattr、inode、lsof 在 macOS 真机通过。
7. 关闭窗口后应用继续后台运行，托盘退出才真正结束。
8. DMG 安装后的应用能启动、登录并完成同步冒烟测试。
