## 3. P0 阻断项

### 3.1 应用入口与 UI（P0/P5 已修复）

`Main.kt` 创建真实 Composition Root，并提供恢复登录、文件浏览、传输、六分区设置与日志路由。窗口关闭只隐藏，托盘退出才执行 3.2 秒优雅关闭。

### 3.2 OAuth 与 token 合同（P1 已修复）

登录已实现 PKCE/state、系统浏览器、127.0.0.1 loopback、state 校验、取消/超时关闭；token refresh 为真 singleflight，token.bin 原子 0600 保存且 IOPlatformUUID 缺失时 fail closed。OAuth 切换账号后会先停旧运行时、清 DB/checkpoint 并重置挂载配置，不再复用上一账号的目录与基线。

### 3.3 Drive DTO 与严格协议（P1 已修复）

`DriveFile`、列表页、单文件、Changes 已使用严格解析，兼容字段别名、数值类型、hash 别名、四种文件夹 MIME 和单 parent 约束。

### 3.4 Files/Changes/About/Thumbnail 合同（P1 基本完成）

- list/search 使用真实 query DSL 和完整 URL 编码
- create/update 的 JSON 使用 ASCII-only 转义，multipart metadata 使用原始 UTF-8
- create/rename/move/delete 均核验 HTTP/File 语义，不确定响应通过 GET/list 收敛
- 直接 rename/move/delete 与同步周期互斥；已核验远端写入后原子迁移本地子树与 DB 基线，delete 保留墓碑并写 Completed 留痕
- Changes 400/410 已分类为 `ChangesCursorInvalid`，并已接入“保留旧 checkpoint → startCursor → BFS → replay → 原子提交”的全量回退

### 3.5 安全上传下载（P4 已完成）

- 三条上传/下载命令与同步周期统一进入持久 TaskRunner，不再存在一次性读取大文件或只查元数据的旁路
- 小文件支持 multipart Create/Update；大文件支持 session/offset/source snapshot 持久化、分片 PUT、308 `rangeList` 和最终状态轮询
- 不确定写只用服务端状态/persisted id/只读查重确认，Update 永不降级 Create
- 下载支持 Range、sidecar、416 单次回退、1MiB 流式写入与 SHA-256、二次元数据核验、fsync 和原子安装

### 3.6 数据库首次启动与迁移（P0 已修复）

新库现通过 SQLDelight `Schema.create` 直接建立 v6 终态 schema；旧库支持 v2/v3/v4/v5 逐级升级并写入 `PRAGMA user_version=6`。

`transfer_queue` 已补齐 operation、error_kind、next_retry_at、source snapshot、resume 上下文、remote result 等终态字段和索引；CAS 使用 SQLite 受影响行数，迟到进度只在 Running 状态生效。

### 3.7 同步引擎信任门（P3 已完成）

云树现固定执行 getStartCursor → BFS → Changes replay，全量与增量只在完整验证后原子提交；cursor 失效、分页中断、根目录歧义和 checkpoint 写入失败均 fail closed 或回滚。

`CommandService.create()` 已装配 `JvmSyncRuntime`，startup/watcher/manual/timer/recovery 统一提交给唯一周期 owner。临时目录 + Mock Drive 已完成本地新增上传、云端新增占位、DB baseline 和可信 checkpoint 的纵向测试。

挂载目录切换会先停 watcher、封闭新周期、等待已有活动结算，再清理挂载绑定 DB 表与旧/新 checkpoint。从有基线目录切到空目录的集成测试确认远端 delete 调用为 0。`~/` 由唯一 JVM 路径解析器展开，并禁止把用户主目录本身设为挂载根。

P4 已将覆盖上传、完整下载、破坏性删除、移动和冲突副本接回唯一同步 owner，并同步结算 checkpoint 与 baseline。

### 3.8 本地挂载和监听（P2 已完成）

已实现递归扫描、真实 inode、inode move detection、JNA xattr/FSEvents、FinderInfo 灰标、debounce/warmup/generation、lsof 双重检查、安全 PlaceholderManager 和流式 hasher。这些平台能力已有单元与 macOS 真机测试，扫描、watcher、上传稳定性和可恢复传输均接入唯一同步周期。

### 3.9 释放空间安全合同（P4 已完成）

单项和批量释放空间均进入 `JvmFreeUpService`：可信云树、双重本地/DB snapshot、远端版本核验、write-ahead staging、占位符、baseline CAS、inode 更新和启动恢复均已接入；原路径出现新用户文件时保留 staging 并拒绝覆盖。

### 3.10 macOS 生命周期（P5 已完成）

文件锁+loopback 单实例、Compose SystemTray/NSStatusItem、regular/accessory 切换、Dock/Cmd+Q 拦截、Apple Event 系统退出识别、LaunchAgent bootstrap/bootout 和 incomplete 哨兵均已装配。打包 `.app` 的隐藏启动和第二实例冒烟已通过；Close/Cmd+Q/Dock Quit/系统关机/登出/LaunchAgent 仍需在最终 DMG 上人工验收。

### 3.11 更新、打包与旧版兼容（P6 自动化已完成）

版本由 `gradle.properties` 单一派生；DMG 沿用原 bundle id、图标与数据目录，配置最低 macOS 12.0、JVM/JNA entitlements 和 `java.sql` 运行时模块。等价更新器保留启动/定时/聚焦节流/手动检查与主界面更新提示，并执行 HTTPS manifest、版本与系统约束、传输等待、SHA-256、codesign、Gatekeeper、Team ID、staging/helper 与回滚。

CI/Release workflow 已覆盖 317 个测试、DMG、打包应用冒烟、Developer ID/notary/staple 门禁和更新 manifest。旧 `token.bin`、camelCase `config.json`、数据库 v2–v6 由兼容测试覆盖。正式签名链和真实用户数据只允许在具备 secrets 的 Release Candidate 环境验收。

## 6. 完成判定

只有满足以下条件，项目才可从“原型”进入“可用 Alpha”：

1. 应用启动完成数据库、配置、日志、网络、服务和引擎装配。
2. 真实 OAuth 可以登录、恢复、刷新、取消和退出。
3. Mock HTTP 合同覆盖全部华为 API 怪癖和写后核验。
4. 临时目录中能完成一次本地新增上传、云端新增占位和按需下载。
5. 重启后能恢复云树 checkpoint 和中断传输，不重复创建远端文件。
6. FSEvents、xattr、inode、lsof 在 macOS 真机通过。
7. 关闭窗口后应用继续后台运行，托盘退出才真正结束。
8. DMG 安装后的应用能启动、登录并完成同步冒烟测试。
