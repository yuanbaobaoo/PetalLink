## 1. 产品定位

**PetalLink** —— 华为云盘 macOS 客户端。

- 华为云空间官方未提供 macOS 版本，Mac 用户只能通过浏览器访问 cloud.huawei.com，无法与本地文件系统集成。
- PetalLink 通过**华为 Drive REST API 直连**云盘，不依赖 HMS Core SDK（macOS 上不存在）。
- 核心思路：把云盘内容映射到本地目录，在 Finder 中直接操作，本地与云端双向同步。
- **产品类型**：企业级工具类桌面应用，视觉风格为品牌蓝（logo 主色 #0053DB）× macOS 原生（v2 重设计，原型见 `design/`，设计 token 详见 `docs/plan/09-设计系统.md`）。

| 项 | 值 |
|---|---|
| 产品名 | PetalLink |
| Bundle ID（正式） | `io.github.yuanbaobaoo.PetalLink` |
| Bundle ID（dev） | `io.github.yuanbaobaoo.PetalLink-dev`（隔离数据目录/单实例锁） |
| 窗口标题 | `PetalLink - 华为云盘客户端开源版` |

## 4. 应用启动流程

> 源码：`src/lib.rs::run()` + `src/main.rs`

### 4.1 装配顺序（lib.rs::run）

```
init_logger()          → 三层日志（stdout + 滚动文件 + 环形缓冲 1000 条）
load_env()             → 加载 .env，解析 client_id / client_secret
tauri::Builder
  .plugin(single_instance)  → ★ 最先注册：第二进程 exit(0) + 聚焦（防双 FSEvents watcher）
  .plugin(shell/dialog/fs/notification/os/process/updater)
  .invoke_handler(49 个命令)
  .on_window_event()        → 关窗拦截（prevent_close + hide + accessory）；获焦切回 regular
  .setup():
    1. init_activation_policy()         → --hidden 检测，设 regular/accessory
    2. install_terminate_interceptor()   → ★ swizzle NSApplication terminate:（拦截 Dock/Cmd+Q）
    3. 检测 last_crash.marker            → 崩溃标记日志后清理
    4. tray::setup()                     → 创建 NSStatusItem 托盘
    5. Login Items 去重                  → 若 LaunchAgent 已启用，移除 Login Items 重复项
    6. 显示/隐藏窗口                     → 手动启动 show+focus；--hidden 保持隐藏
    7. 加载 config
    8. 孤儿状态清理                      → token 丢失但 config 残留 → cleanup_orphan_state
    9. 若 mount_configured → MountManager + SyncEngine 初始化
  .build()
app.run()               → ExitRequested 拦截（accessory）；Exit 时 net_guard shutdown + flush
```

### 4.2 用户流程

1. 启动 App → 登录页 → 点「使用华为账号登录」→ 浏览器打开华为授权页
2. 浏览器完成授权（账号密码或扫码）→ 自动回到 App（OAuth 回调 127.0.0.1:9999）
3. **主界面顶部出现「尚未配置同步目录」提示条** → 点「选择目录」→ 选一个空目录
4. 提示条变为「同步索引」按钮 → 点击开始首次云端索引拉取（BFS 拉取全量文件树到本地）
5. 完成后自动进入「双端对齐」模式：本地文件变更实时上传，云端变更通过手动/定时刷新同步

### 4.3 后台运行机制

- 启动后系统菜单栏右上角出现云朵图标，悬停显示「PetalLink — 后台同步中」
- 关闭窗口 / ⌘W / ⌘Q → **仅隐藏 UI 层**（窗口 + Dock 图标消失），同步引擎继续运行
  - 通过 swizzle `-[NSApplication terminate:]` 实现：拦截 Dock/Cmd+Q，改为 hide + accessory
  - 系统关机/登出的 Apple Event（`kAEQuitApplication`）放行真正退出
- 点击菜单栏图标 →「显示主窗口」→ UI 恢复（accessory 切回 regular）
- **菜单栏「退出 PetalLink」是真正终止进程的唯一路径**（设 `REAL_QUIT=true`）
- 开机自启：写 LaunchAgent plist，带 `--hidden` 参数（开机只显示菜单栏图标）

## 5. 单实例守护

第二进程启动时直接 exit(0) + 聚焦已运行实例窗口。

**为什么必须**：防止双进程各自创建 FSEvents watcher 监听同一挂载目录 → 互相触发 sync cycle → 基于 stale cloud_tree 误判「本地新建」疯狂上传。

**实现**：`tauri-plugin-single-instance`（socket 由 tauri.conf.json 的 identifier 决定）。第二实例回调中：若带 `--hidden`（LaunchAgent 重复触发）不显示窗口；否则 show + focus + `set_regular()`（accessory 模式恢复可交互）。

> Kotlin 重构需保留等效的单实例机制（文件锁 / 端口占用）。

## 6. 非功能需求

### 性能

| 指标 | 要求 | 实现方式 |
|---|---|---|
| UI 响应 | 任意操作 < 200ms | 后端异步处理，主线程不阻塞 |
| 并发传输 | 默认 6（可配 1-20） | tokio task 并发（Semaphore） |
| 同步扫描 | 1000 文件目录 < 10s | BFS 8 并发 + 云端树磁盘缓存 |
| 内存占用 | 空闲 < 200MB，传输高峰 < 500MB | 编译优化（lto + opt-level=s + strip） |
| 启动速度 | 非首次 < 1s | 云端树缓存 ~200ms（17K 文件） |

### 安全性

| 项 | 要求 |
|---|---|
| Token 存储 | 加密文件（ChaCha20-Poly1305 AEAD，机器码派生密钥，权限 0600） |
| Client 凭据 | 编译期注入（build.rs 从 .env），不硬编码、不提交 git |
| 网络通信 | 全程 HTTPS |
| 本地回调 | 仅绑定 127.0.0.1:9999（不监听 0.0.0.0） |
| 日志 | 不打印完整 token/secret |
| CSP | `default-src 'self'`，connect-src 仅限 `*.huawei.com(.cn)` + GitHub |

### 兼容性

| 项 | 要求 |
|---|---|
| macOS 版本 | 12 Monterey 及以上（tauri.conf.json `minimumSystemVersion: 12.0`） |
| 架构 | Apple Silicon (arm64) + Intel (x86_64) |

### Release 编译优化（Cargo.toml `[profile.release]`）

```toml
panic = "abort"       # 减小二进制
codegen-units = 1     # 最大优化
lto = true            # 链接时优化
opt-level = "s"       # 优化体积
strip = true          # 去符号
```

## 7. 关键设计决策（重构需保留的判断）

| 决策项 | 选择 | 理由 |
|---|---|---|
| OAuth Scope | `openid profile drive`（全盘访问） | `drive.file` 只能看到本应用创建/打开的文件，网页/其他客户端上传的看不到 |
| OAuth 回调 | 127.0.0.1:9999 loopback HTTP | AGC 接受；不用 URL Scheme |
| 云端变更检测 | 定时增量（Changes API）+ 全量 BFS 兜底 | 华为无 webhook |
| 本地变更处理 | FSEvents + 3s debounce + 2s warmup + 稳定性检查 + 配额校验 | 三段式控制，纯事件驱动 |
| 首次同步 | 手动引导（选空目录 → 点「同步索引」） | 避免覆盖用户本地已有内容 |
| 默认挂载目录 | 空（不设默认） | 用户未配置时不上传/同步任何内容 |
| 退出后目录 | 保持可写，启动时按快照自动 diff | 最自然体验 |
| 冲突策略 | 60s 容忍窗口 + 副本去重 + 目录保护 | 本地时钟偏差免疫 |
| 沙盒 | 关闭（macOSPrivateApi: true） | 双向同步需读写用户挂载目录 |
| 中文文件名 | `>0x7F → \uXXXX` ASCII 转义 | 华为 JSON 解析器不接受 UTF-8 直传（仅 application/json；multipart 容忍） |
| 内部文件隔离 | `.hwcloud_` 前缀全局硬编码过滤 | 内部缓存/快照绝不参与同步 |
| 占位状态判据 | 以 `com.hwcloud.state` xattr 为唯一权威 | 文件大小/文件名/Finder 标签不参与判定 |
| 网络状态 | 连续 2 次探测成功才转 Online | 防抖动，离线快上线慢 |

## 8. 构建链路

### 8.1 build.rs（构建脚本）

> 源码：`build.rs`

1. **凭据注入**（最早阶段，缺失则 panic 阻断编译）：
   - 优先级：已显式设置的环境变量 > `.env` 文件
   - 读取 `HWCLOUD_CLIENT_ID` / `HWCLOUD_CLIENT_SECRET` → `cargo:rustc-env` 注入 → 编译期 `option_env!` 可见
   - `.env` 不存在或凭据为空 → panic 阻断
2. `tauri_build::build()`：读取 tauri.conf.json 生成构建期常量
3. **图标同步**：`assets/` → `icons/`（PNG 复制 + `iconutil` 生成 `.icns`），使 `assets/` 成为唯一图源

### 8.2 tauri.conf.json 关键配置

| 项 | 值 | 说明 |
|---|---|---|
| `identifier` | `io.github.yuanbaobaoo.PetalLink` | 决定数据目录/单实例锁 |
| `app.windows[0]` | 1280×800, min 700×480 | `visible: false`（手动 show 防闪现） |
| `titleBarStyle` | `Visible` + `hiddenTitle: false` | `trafficLightPosition: {x:16, y:18}` |
| `macOSPrivateApi` | `true` | 关闭沙盒，支持双向同步 |
| `security.csp` | `default-src 'self'` | connect-src 限华为域名 + GitHub |
| `bundle.macOS.minimumSystemVersion` | `12.0` | Monterey 及以上 |
| `bundle.createUpdaterArtifacts` | `true` | 生成 `.app.tar.gz` + `.sig` |
| `plugins.updater` | Ed25519 pubkey + GitHub Release endpoint | 自动更新 |

### 8.3 凭据三级解析（constants.rs）

优先级（高 → 低）：
1. **构建期环境变量**（`HWCLOUD_CLIENT_ID` / `HWCLOUD_CLIENT_SECRET`，由 build.rs 从 .env 注入）
2. **运行时 .env 文件**（dotenvy 加载，启动期 `set_env_secret` / `set_env_client_id` 写入 OnceCell）
3. **占位符**（`PLACEHOLDER_SECRET = "REPLACE_WITH_REAL_SECRET"`，登录会被拒）

- `resolved_client_id()` / `resolved_client_secret()`：合并优先级返回最终值
- `client_id_configured()` / `client_secret_configured()`：UI 登录按钮可点判断
- **无硬编码默认值**，必须由用户显式提供
