### 华为 API 编码注意事项（高频踩坑）

| 问题 | 根因 | 修复 |
|---|---|---|
| Scope `/` 被编码 → invalid scope 1101 | scope 空格表示分隔，`/` 不编码 | scope 原始字符串拼接，空格用 `%20` |
| Authorization code 的 `+` 被当空格 → invalid code 1101 | form-urlencoded 标准行为 | 手工拼 form body，精确 percent-encoding |
| Client 凭据注入 | 不能硬编码/提交 git | 编译期/运行时从配置注入 |

### F-FILE-09 释放空间 13 步 TOCTOU 防误删流程

完整流程：双重本地/DB 复核 + 远端核验夹中间 + 原子 staging + CAS 回滚，任何一步失败即拒绝 unlink。

- **第 1 步 本地复核**：`symlink_metadata` 确认非占位符（`com.hwcloud.state != placeholder`），读 `source_mtime` / `source_size`
- **第 2 步 DB 复核**：无活动传输任务；`file_id` 与 `local_path` 匹配；`status = SYNCED`；且 5 字段一致（file_id / local_path / local_mtime / local_size / status）
- **第 3 步 云树复核**：`cloud.get(rel).id == file_id`
- **第 4 步 远端核验**：`FILES_API.get(file_id)` 返回的 id / size / editedTime 一致；并 `verify_deleted` 确认未被回收
- **第 5 步 二次本地复核**：远端核验期间本地可能变化，重新读 `symlink_metadata` / mtime / size
- **第 6 步 二次 DB 复核**：5 字段（file_id / local_path / local_mtime / local_size / status）全部未变
- **第 7 步 原子 staging**：写 `XATTR_FREE_UP_RELATIVE_PATH`（`com.hwcloud.freeUpRelativePath`）→ rename 到 `.hwcloud_freeup-` 前缀暂存文件
- **第 8 步 重建占位**：`create_placeholder_strict`（严格版，不做存在性检查，直接 `create_new`）
- **第 9 步 DB CAS**：`UPDATE ... WHERE ... AND status = SYNCED AND local_mtime = ? AND local_size = ?`（Compare-And-Set 原子推进）
- **失败回滚**：`restore_staged_free_up`（还原暂存文件）+ `rollback_free_up_baseline`（反向 CAS 回滚基线）

## 文件编辑期间同步（三段式控制）

这是同步引擎的核心防抖逻辑，**重构必须完整保留**：

**第一阶段：变更捕获（Debounce）**
- FSEvents 监听文件变化 → 启动 3s 去抖计时器
- 每次新变化重置计时器
- 到期才进入下一阶段

**第二阶段：稳定性检查（Stability Check）**
- mtime > 5s + size 稳定 3s + lsof 双重检查（间隔 1s 两次）
- 任一不通过则重新等待
- lsof 白名单（10 进程）：`mds`、`mdworker_shared`、`mdimport`、`mdflagworker`、`QuickLookSatellite`、`qlmanage`、`corespotlightd`、`secd`、`bird`、`CoreServicesUIAgent`（Spotlight / QuickLook / 安全守护等只读系统进程）

**第三阶段：上传前配额校验**
- 每次上传前 GET `/drive/v1/about` 获取 usedSpace 和 userCapacity
- 文件大小 > remaining → 暂停上传，状态置「失败：配额不足」

### 特殊场景

| 场景 | 处理 |
|---|---|
| 文件被持续编辑 > 5min | 标记 Editing，跳过自动同步；可手动触发上传 |
| 编辑期间 App 退出 | 下次启动重新触发 debounce + 稳定性检查 |
| 文件改名 | 检测到 rename 事件 → 同步改名到云端 |
| App 退出后用户修改文件 | `.hwcloud_syncstate` 快照启动时 diff 自动同步 |
| 复制真实文件（带 xattr，源路径仍存在） | 检测到相同 fileId 时核验旧路径仍携带同一 fileId → 判定为复制（非改名） |
